gdp-setup

Initialize and remember the GDP runtime environment.

Documentation Tree

  • Main documentation
  • gdp-setup: initialize and remember runtime and source-data locations.
  • gdp-util: read metadata from CASA gain, bandpass, or MeasurementSet tables.
  • gdp-stats: compute antenna-wise gain and bandpass products.
  • gdp-plot: plot GDP NPZ products and create missing products with gdp-stats.
  • gdp-plan-run: run GDP command intents from a plan file.

Product Formats

  • gains NPZ
  • stats NPZ
  • KS NPZ
  • self-corr NPZ
  • cross-corr NPZ

Purpose

gdp-setup creates the runtime directory structure and saves separate gain and bandpass table locations so later GDP commands can reuse the correct default for each mode.

Usage

script/gdp-setup --gain-table /path/to/gain/table.g --bandpass-table /path/to/bandpass/table.b --rundir /path/to/work/rundir  [save separate gain and bandpass defaults plus runtime directory]
script/gdp-setup --source-data /path/to/source/data  [save a general source-data directory]
script/gdp-setup --show  [print the saved GDP setup]

Options

OptionDescription
--rundir PATHRuntime directory to create or use. Defaults to the saved value, then <GDP>/rundir.
--source-data PATHSource-data directory remembered for later GDP commands.
--gain-table PATHGain calibration table remembered as the default for gain-mode GDP commands.
--bandpass-table PATHBandpass calibration table remembered as the default for bandpass-mode GDP commands.
--gain-dir PATHBackward-compatible alias for --gain-table.
--config PATHConfiguration JSON path. Defaults to <GDP>/.gdp-config.json.
--showPrint the saved configuration and exit.
--dry-runShow the directories and configuration that would be written.
--allow-missing-sourceAllow source, gain, or bandpass paths to be recorded before they exist.
