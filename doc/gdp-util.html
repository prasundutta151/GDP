Purpose

gdp-util provides small metadata utilities behind one command. It can print table headers, observing dates, channel-width summaries, and the saved GDP setup configuration. It also provides --git-push for the GDP release/git workflow.

Usage

script/gdp-util /path/to/table.g --header  [show table rows, columns, IDs, and data-column descriptions]
script/gdp-util /path/to/table.g --header --keywords --json  [write header and keyword metadata as JSON]
script/gdp-util /path/to/table.g --date  [show observing date and time span]
script/gdp-util /path/to/table.ms --channel-width  [show channel-width statistics in MHz]
script/gdp-util --config  [print the saved GDP setup configuration]
script/gdp-util --git-push -m "Describe the change"  [build the version archive, commit, and push]

Options

OptionDescription
input_tableOptional positional CASA table or MeasurementSet path.
--input-table PATHNamed input path. Overrides the positional path.
--headerPrint row count, columns, common IDs, data columns, and column descriptions.
--keywordsInclude CASA table keywords in header output.
--datePrint IST observing dates and time span from the TIME column.
--channel-widthPrint channel-width statistics in MHz.
--configPrint the saved GDP setup configuration.
--git-pushCreate gdp-<VERSION>.tgz, stage GDP source/docs/notes/archive files, commit, and push the current branch.
-m, --message TEXTCommit message for --git-push. Defaults to GDP update <version>.
--no-pushWith --git-push, create the archive and commit without pushing.
--jsonWrite machine-readable JSON output.

Git Workflow

The --git-push mode builds a versioned archive from the GDP tree, excluding local runtime products, git internals, cache files, and local setup state. It stages project files such as DEV_NOTES.md, DEV_PLAN.md, REQUIREMENTS.md, doc/, script/, and the version archive before committing and pushing.
