gdp-plot

Plot GDP NPZ products, creating missing products with gdp-stats when needed.

Documentation Tree

  • Main documentation
  • gdp-setup: initialize and remember runtime and source-data locations.
  • gdp-util: read metadata from CASA gain, bandpass, or MeasurementSet tables.
  • gdp-stats: compute antenna-wise gain and bandpass products.
  • gdp-plot: plot GDP NPZ products and create missing products with gdp-stats.
  • gdp-plan-run: run GDP command intents from a plan file.

Product Formats

  • gains NPZ
  • stats NPZ
  • KS NPZ
  • self-corr NPZ
  • cross-corr NPZ

Purpose

gdp-plot reads the NPZ products created by gdp-stats and saves diagnostic plots under the configured runtime plot directory. If a requested NPZ product is missing, it first runs gdp-stats with the matching subtask and then plots the newly created file. For the gain/bandpass colormap only, gdp-stats --gain-colormap can also create the plot directly while producing the gains NPZ.

Usage

script/gdp-plot --mode gain -pmode stats  [plot the gain stats NPZ, creating it first if missing]
script/gdp-plot --mode gain -pmode stats --combine-scans  [plot one combined allscans stats product]
script/gdp-plot --mode bandpass -pmode gain-colormap --bchan 0 --echan 128  [plot bandpass Real-1/Imag percent colormaps by channel]
script/gdp-plot --mode gain -pmode gain-hist --hist-bins 64  [plot AntStat-style per-antenna Real-1/Imag histograms from gains NPZ]
script/gdp-plot --mode gain -pmode antenna 0,1,2  [make one AntStat-style gain-time plot per antenna, with Stokes side by side]
script/gdp-plot --mode gain -pmode self-corr-antenna --antenna 0,1,2  [make per-antenna self-correlation plots]
script/gdp-plot --mode both -pmode gain-colormap,gain-hist,stats,ks  [make selected plot modes for gain and bandpass]
script/gdp-plot -pmode cross-corr-colormap --antenna 0,1,2  [plot selected antennas from cross-corr NPZ]

Options

OptionDescription
--mode {gain,bandpass,both}Select which mode to plot. both runs each selected plot task for gain and bandpass.
-pmode MODE[,MODE...]Select plot products. Supported modes are gain-colormap, gain-hist, antenna, stats, ks, self-corr-colormap, self-corr-antenna, and cross-corr-colormap.
gain-colormapPlot AntStat-style Real-1 and Imag percent colormaps from gdp-gains-<mode>-<scan>.npz. Gain-mode plots use elapsed seconds from the saved time and interval arrays; bandpass plots use channel.
gain-histPlot AntStat-style per-antenna histograms from the full gains NPZ samples. Panels are Stokes by Real-1/Imag, y-axis is gain value percent, and color is log10 percentage per antenna.
antennaMake one AntStat-style plot per selected antenna, with Stokes side by side and Real-1/Imag stacked. Gain mode plots versus elapsed seconds; bandpass mode plots versus channel. Antennas may be supplied directly after the mode, for example -pmode antenna 3,5, or with --antenna.
statsPlot AntStat-style 2x2 per-antenna statistics grids from the stats NPZ product: mean, std, skewness, and kurtosis.
ksPlot AntStat-style per-antenna KS D-statistic lines from the KS NPZ product.
self-corr-colormapPlot AntStat-style structure-function colormaps with Stokes rows, Real-1/Imag columns, antenna on x, tau on y, and log10(S2) color.
self-corr-antennaPlot AntStat-style per-antenna structure-function curves with S2=1 and threshold reference lines.
cross-corr-colormapPlot antenna-pair tcorr colormaps from the cross-correlation NPZ product.
--combine-scansPlot one combined allscans-style product. Default: false, so GDP makes one plot per scan.
--hist-bins INTNumber of gain-value bins for --gain-hist. Default: 64.
--hist-range MIN,MAXOptional gain-value range in percent for --gain-hist. Default: symmetric range covering the plotted extrema.
--antenna LISTComma-separated antenna numbers to plot. The same list is passed to gdp-stats if missing products must be created.
--scan LISTUse the matching scan-specific product name, and pass the scan list to gdp-stats if products are missing.
--input-table PATHExplicit input table passed to gdp-stats when an NPZ product must be created.
--gain-table PATH, --bandpass-table PATHMode-specific tables passed to gdp-stats when missing gain or bandpass NPZ products must be created.
--gain-dir PATHBackward-compatible alias for --gain-table.
--bchan INT, --echan INTBandpass channel selection passed to gdp-stats when needed.
--s2-thr FLOATThreshold used for self-correlation reference lines and colormap markers. Default: 0.9.
--plot-dir PATHOutput plot directory. Default: configured plots/<format> runtime directory.
--format {png,pdf,eps}Plot file format. Default: png.
--recomputeRun gdp-stats before plotting even when the NPZ file already exists.
--dry-runPrint the gdp-stats commands that would create missing products.

Default Plot Names

Plots are saved as gdp-plot-<task>-<mode>-<scan>.<format>. By default the scan field is per scan, such as scan1. With --combine-scans, the scan field is allscans when no scan list is supplied.
