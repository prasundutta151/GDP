gdp-stats

Compute antenna-wise gain and bandpass statistics products.

Documentation Tree

  • Main documentation
  • gdp-setup: initialize and remember runtime and source-data locations.
  • gdp-util: read metadata from CASA gain, bandpass, or MeasurementSet tables.
  • gdp-stats: compute antenna-wise gain and bandpass products.
  • gdp-plot: plot GDP NPZ products and create missing products with gdp-stats.
  • gdp-plan-run: run GDP command intents from a plan file.

Product Formats

  • gains NPZ
  • stats NPZ
  • KS NPZ
  • self-corr NPZ
  • cross-corr NPZ

Purpose

gdp-stats reads a CASA gain or bandpass table and creates GDP data products. If no input table is supplied, it uses the saved gain_dir from gdp-setup. Each requested subtask writes to the runtime product folders and records timing information in a log file.

Usage

script/gdp-setup --gain-dir /path/to/table.g --rundir /path/to/rundir  [save the default input table and runtime directory]
script/gdp-stats --stats --csv  [compute statistics from the saved input and also write CSV]
script/gdp-stats --mode gain --gain-colormap  [create gains NPZ and save a gain amplitude colormap plot]
script/gdp-stats --stats --combine-scans  [combine scans into one allscans stats product]
script/gdp-stats /path/to/table.b --mode bandpass --bchan 0 --echan 128 --gains --stats  [process selected bandpass channels and save gains plus stats]
script/gdp-stats --scan 1,2 --antennas 0,1,2 --all  [run every stats subtask for selected scans and antennas]

Options

OptionDescription
input_tableOptional positional CASA gain/bandpass table path.
--input-table PATHNamed input table path. Overrides the positional path.
--gain-dir PATHNamed gain/bandpass table path. Defaults to the saved gdp-setup gain directory when omitted.
--mode {auto,gain,bandpass}Statistics mode. Default: auto.
--scan LISTComma-separated scans. Default: all scans.
--combine-scansCombine selected scans into one allscans-style product. Default: false, so GDP writes one product per scan.
--antennas LISTComma-separated antennas. Default: all antennas.
--bchan INTStart channel, inclusive.
--echan INTEnd channel, exclusive.
--use-flags, --no-use-flagsApply or ignore the CASA FLAG mask. Default: apply flags.
--subtract-mean, --no-subtract-meanSubtract real and imaginary means before computing statistics. Default: enabled.
--gainsSave selected gain/bandpass samples as a GDP gains NPZ product.
--gain-colormapAlso save a gain/bandpass amplitude colormap plot from the gains NPZ product. Default: off.
--statsCompute mean, std, skew, kurtosis, median, MAD, min, and max statistics. This is the default when no subtask is selected.
--ksCreate the KS product schema. Full sample KS calculation is planned for a later implementation pass.
--self-corrCompute self structure-function correlation curves and save a long-format NPZ product.
--cross-corrCompute cross structure-function correlation times between antenna/stokes/component pairs and save NPZ output.
--allRun all GDP stats subtasks.
--output-npz PATHOutput NPZ path. Default: configured stats/npz runtime directory.
--output-gains-npz PATHOutput gains NPZ path. Default: configured gains/npz runtime directory.
--output-ks-npz PATHOutput KS NPZ path. Default: configured ks/npz runtime directory.
--output-self-corr-npz PATHOutput self-correlation NPZ path. Default: configured self-corr/npz runtime directory.
--output-cross-corr-npz PATHOutput cross-correlation NPZ path. Default: configured cross-corr/npz runtime directory.
--output-cross-corr-csv PATHOptional cross-correlation CSV path. Implies CSV output for --cross-corr.
--output-gain-colormap PATHOutput path for the optional gain colormap plot.
--plot-format {png,pdf,eps}Plot format for --gain-colormap. Default: png.
--corr-threshold FLOATS2 threshold used for cross-correlation time extraction. Default: 0.7.
--corr-nbin INTStructure-function lag bin count. Default: 32.
--corr-bintype {log,lin}Structure-function lag binning. Default: log.
--corr-njack INTJackknife folds for structure-function errors. Default: 32.
--csvAlso write optional CSV products for selected subtasks that support CSV, including stats and cross-correlation.
--output-csv PATHWrite CSV to a specific path; implies --csv.

Subtasks and Logs

Each subtask prints a start/done message with elapsed time and writes the same messages to a timestamped log file under the configured logs runtime directory.

The <scan> field is scan1, scan2, etc. by default. With --combine-scans, the field is allscans when no scan list is supplied, or scan1-2/scan5 style when --scan is supplied.
