GDP Step-by-Step Guide

A practical workflow for setting up GDP, generating data products, making plots, checking outputs, and preparing a release.

Documentation Tree

  • Main documentation
  • Step-by-step guide: run GDP from setup through products, plots, and release steps.
  • gdp-setup: initialize and remember runtime and source-data locations.
  • gdp-util: read metadata from CASA gain, bandpass, or MeasurementSet tables.
  • gdp-stats: compute antenna-wise gain and bandpass products.
  • gdp-plot: plot GDP NPZ products and create missing products with gdp-stats.
  • gdp-plan-run: run GDP command intents from a plan file.

Product Formats

  • gains NPZ
  • stats NPZ
  • KS NPZ
  • self-corr NPZ
  • cross-corr NPZ

Workflow Overview

GDP is normally used in this order: set up the table and runtime paths, inspect the input table, generate NPZ products, make plots, review the outputs, then optionally update the version and push a release.

StepGoalMain Command
1Remember input and runtime paths.script/gdp-setup
2Check table metadata before processing.script/gdp-util --header
3Create stats and KS products.script/gdp-stats --stats --ks
4Create self-correlation products.script/gdp-stats --self-corr
5Make diagnostic plots.script/gdp-plot -pmode ...
6Run a pipeline plan when repeated operations are needed.script/gdp-plan-run
7Version and release the software.script/gdp-util --version --git-push

1. Set Up GDP

Run setup once for a project. GDP then remembers the gain table, optional bandpass table, and runtime directory for later commands.

cd /Users/prasun/Documents/GDP
script/gdp-setup --gain-table /path/to/gain/table.g --bandpass-table /path/to/bandpass/table.b --rundir /path/to/GDP_RUN

Check the saved setup:

script/gdp-setup --show

Expected runtime output pattern:

GDP setup
gain_table: /path/to/gain/table.g
bandpass_table: /path/to/bandpass/table.b
rundir: /path/to/GDP_RUN
data-product directories: created or already present
plots directories: created or already present

2. Inspect The Table

Use gdp-util to verify scans, antennas, columns, and observing time coverage before generating products.

script/gdp-util --header
script/gdp-util --date
script/gdp-util --channel-width

Expected runtime output pattern:

header
======
path: /path/to/gain/table.g
nrows: ...
columns: [...]
data_columns: [...]
scans: [...]
antennas: [...]

3. Generate Stats And KS Products

Run these in a CASA-enabled shell because table reading uses casatools.

casa --nogui --nologger --nologfile --log2term -c script/gdp-stats --mode gain --scan 2 --stats --ks

Expected runtime output pattern:

START stats scan2
Wrote stats NPZ: .../data-product/stats/npz/gdp-stats-gain-scan2.npz
DONE  stats scan2 elapsed_sec=...
START ks scan2
Wrote KS NPZ: .../data-product/ks/npz/gdp-ks-gain-scan2.npz
DONE  ks scan2 elapsed_sec=...

Product format details are in stats NPZ and KS NPZ.

4. Generate Self-Correlation Products

The self-correlation product should normally be generated for all antennas unless a focused antenna subset is required.

casa --nogui --nologger --nologfile --log2term -c script/gdp-stats --mode gain --scan 2 --self-corr

Expected runtime output pattern:

START self-corr scan2
SKIP self-corr scan=2 antenna=... reason=fewer than two finite unflagged samples
Wrote self-corr NPZ: .../data-product/self-corr/npz/gdp-self-corr-gain-scan2.npz
DONE  self-corr scan2 elapsed_sec=...

Skipped antenna/stokes/component rows are kept as NaN lag bins so the corresponding colormap cells remain white.

5. Make Plots

Once products exist, plotting can run from the normal shell. Use --recompute only when the shell can also run gdp-stats with CASA access.

script/gdp-plot --mode gain --scan 2 -pmode stats,ks
script/gdp-plot --mode gain --scan 2 -pmode self-corr-colormap
script/gdp-plot --mode gain --scan 2 -pmode gain-colormap,gain-hist
script/gdp-plot --mode gain --scan 2 -pmode antenna 2

Expected runtime output pattern:

Using existing stats NPZ: .../gdp-stats-gain-scan2.npz
START plot stats mode=gain scan=scan2
Wrote plot: .../plots/png/gdp-plot-stats-gain-scan2.png
DONE  plot stats elapsed_sec=...
PlotSampleWhat To Check
Statssample-stats-gain-scan2.pngAll selected antennas appear on the x-axis.
KSsample-ks-gain-scan2.pngD-statistic trends are shown per antenna and Stokes/component.
Self-corr colormapsample-self-corr-colormap-gain-scan2.pngWhite columns or cells correspond to NaN/insufficient samples.
Gain colormapsample-gain-colormap-gain-scan2.pngReal-1 and Imag panels show antenna/time gain structure.

6. Run A Plan

Use plan files for repeatable processing. Start with a dry run, then run the plan after checking commands.

script/gdp-plan-run pipelines/sample_stats.plan --dry-run
script/gdp-plan-run pipelines/sample_stats.plan

Expected runtime output pattern:

PLAN variable ...
DRY RUN gdp-stats ...
DRY RUN gdp-plot ...
DONE plan elapsed_sec=...

7. Version And Release

Use gdp-util to bump the version and then push. When --version and --git-push are used together, the version update happens first.

script/gdp-util --version --git-push -m "Release update"
script/gdp-util --version main --git-push -m "Major release"
script/gdp-util --version 1.2.0 --git-push -m "Release 1.2.0"

Expected runtime output pattern:

VERSION updated to ...
Staged/current status:
...
Committed changes on main.
Pushed main to origin.
Version archive: .../gdp-...tgz

Update Policy

This guide should be updated whenever GDP gains a new command, product format, plot mode, runtime convention, or release step. Add new steps here when a workflow becomes stable enough to recommend.
