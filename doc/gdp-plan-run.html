gdp-plan-run

Run repeatable GDP workflows from simple plan files.

Documentation Tree

  • Main documentation
  • gdp-setup: initialize and remember runtime and source-data locations.
  • gdp-util: read metadata from CASA gain, bandpass, or MeasurementSet tables.
  • gdp-stats: compute antenna-wise gain and bandpass products.
  • gdp-plot: plot GDP NPZ products and create missing products with gdp-stats.
  • gdp-plan-run: run GDP command intents from a plan file.

Product Formats

  • gains NPZ
  • stats NPZ
  • KS NPZ
  • self-corr NPZ
  • cross-corr NPZ

Purpose

gdp-plan-run reads a plan file, expands internal variables, and runs GDP command intents such as gdp-stats and gdp-plot. It is the first pipeline runner for repeatable GDP workflows.

Usage

script/gdp-plan-run pipelines/sample_stats.plan  [run every intent in the plan]
script/gdp-plan-run pipelines/sample_stats.plan --dry-run  [print commands without executing them]
script/gdp-plan-run pipelines/sample_stats.plan --only gdp-stats  [run only intents whose command is gdp-stats]

Options

OptionDescription
plan_fileRequired GDP plan file path.
--dry-runPrint commands without running them.
--only VALUERun only the matching intent index, command name, or intent name.

Plan Format

A plan can define variables, then one or more intent blocks. Values may reference variables with ${name}. Saved gdp-setup paths, such as ${gain_table}, ${bandpass_table}, ${source_data}, ${stats_npz_dir}, and ${stats_csv_dir}, are available when configured.

variables:
  stats_npz: ${stats_npz_dir}/sample_gain_stats.npz

intent: gdp-stats
  name: gain statistics
  mode: gain
  combine-scans: true
  stats: true
  output-npz: ${stats_npz}
  csv: true
end

intent: gdp-plot
  name: plot gain statistics
  mode: gain
  combine-scans: true
  stats: true
end

Intent keys become CLI flags. For example, output-npz becomes --output-npz. Boolean true values become standalone flags, while false and null values are omitted.
